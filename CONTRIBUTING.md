# Contributing

Thank you for your interest in improving this project.

## Development workflow
1. Fork the repository.
2. Create a feature branch.
3. Keep changes focused on a single feature or bug fix.
4. Update documentation when behavior changes.
5. Ensure Docker-based development environment still works.
6. Open a Pull Request with a clear description.

## Commit Messages
Follow Conventional Commits where practical:
- feat:
- fix:
- docs:
- refactor:
- test:
- chore:

## Code Quality
- Prefer small, reviewable changes.
- Keep architecture consistent with ADR documents.
- Add tests for new functionality when possible.
- Avoid introducing breaking API changes without documentation.
