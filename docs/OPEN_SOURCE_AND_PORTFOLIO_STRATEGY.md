# Open Source and Portfolio Strategy

## Objective
Build a GitHub profile that demonstrates practical engineering ability, consistent open-source participation, and portfolio projects that are relevant to enterprise clients.

## Primary Outcomes

1. Earn trust through merged open-source pull requests.
2. Maintain a small number of high-quality personal repositories.
3. Present clear, client-facing project narratives.
4. Connect GitHub activity to freelancing and international opportunities.

## Open Source Track

### Current Focus
- Start with small, reviewable documentation fixes in established repositories.
- Prioritize repositories related to .NET, AI, observability, and developer tooling.
- Keep each pull request limited to one clear change.

### Target Repositories
- dotnet/docs
- dotnet/aspnetcore
- open-telemetry/opentelemetry-dotnet
- microsoft/semantic-kernel
- qdrant/qdrant

### Pull Request Rules
- One purpose per PR.
- Minimal diff.
- Clear title and summary.
- No unrelated formatting changes.
- Wait for maintainer feedback before opening multiple PRs in the same repository.

## 30-Day Plan

### Week 1
- Keep the first open-source PR clean and responsive.
- Prepare the second open-source issue candidate.
- Improve the flagship project documentation and demo flow.

### Week 2
- Open another small PR after the first one is stable.
- Add a runnable demo path to the flagship project.
- Create a structured backlog for SQL Server Toolkit.

### Week 3
- Start one code-oriented contribution, preferably tests or a small bug fix.
- Add screenshots or demo assets to portfolio repositories.

### Week 4
- Publish a first tagged release for the flagship repository.
- Update the GitHub profile README to highlight open-source participation and flagship projects.

## Success Metrics

- 3 to 5 merged pull requests in 30 days.
- 1 flagship repository with a runnable demo.
- 1 clean GitHub profile README.
- 1 supporting technical toolkit repository started.
- Clear evidence of consistent, professional engineering habits.
