# v0.2.0 — Persistent pgvector and ingestion-job foundation

Released: 2026-07-20

## Summary

Version 0.2.0 moves semantic-index records from process memory into PostgreSQL + pgvector and establishes the durable database contract for future background ingestion. The public upload, search, and ask contracts remain compatible with version 0.1.0.

## Highlights

- persistent `document_chunks` storage using pgvector `vector(8)` values;
- transactional and idempotent PostgreSQL semantic-index upserts;
- cosine-distance search through an HNSW index;
- configuration-driven `InMemory` and `Postgres` semantic-index providers;
- CI verification that retrieval still succeeds after restarting the API container;
- a durable `document_ingestion_jobs` schema with constrained processing states and bounded attempts;
- partial indexes for one active job per document and ordered pending-job claiming;
- expanded .NET, Python, Docker Compose, coverage, CodeQL, and dependency-review validation.

## Migration and local setup

Fresh Docker Compose volumes initialize all required pgvector and ingestion-job objects automatically.

PostgreSQL entrypoint scripts do not rerun for an existing volume. Preserve required data before applying either option:

1. apply the idempotent scripts in `infra/postgres/init` manually through `psql`; or
2. recreate only a disposable local volume with `docker compose down --volumes` followed by `docker compose up --build`.

The deterministic embedding generator currently emits eight-dimensional vectors. A future embedding-dimension change requires an explicit database migration.

## Known limitations

- upload processing is still synchronous;
- the ingestion-job schema exists, but atomic enqueue, the hosted worker, retry execution, and the public processing-status endpoint are follow-up work;
- deterministic embeddings are for reproducible development rather than production retrieval quality;
- FastAPI remains an integration boundary rather than the active extraction or retrieval service;
- authentication, document authorization, tenant isolation, audit logging, and production secret management are not implemented.
