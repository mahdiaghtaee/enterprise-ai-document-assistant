# Security Policy

This project is a portfolio and product-development repository. Security issues should still be treated seriously because the project handles document upload and future authentication workflows.

## Supported Versions

Security fixes are currently applied to the main development branch.

## Reporting a Security Issue

Please do not open a public issue for sensitive security problems.

For now, report security concerns directly to the repository owner through GitHub or the available professional contact channel.

## Security Focus Areas

The most important security areas for this project are:

- File upload validation
- Authentication and authorization
- Access control for documents
- Safe handling of environment variables
- Avoiding secrets in source control
- Input validation for API endpoints
- Secure AI service communication

## Current Status

The project is under active development. Authentication, role-based access, and production hardening are planned milestones.
